# Admin Role Permissions — Foundation & Utility Gates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `is_superuser`-only gating (and, for a few views, no gating at all) on the admin
utility views — data dump, CSV exports, sync gigs, rental/library dashboards, chart import — with
Django/Wagtail Group permissions, and ship the `setup_roles` command that creates those groups.

**Architecture:** Every role is a standard `django.contrib.auth.models.Group`. Views call
`request.user.has_perm('blowcomotion.<codename>')` — Django's `ModelBackend.has_perm` already
returns `True` unconditionally for `is_superuser`, so no view needs a separate superuser check.
Two new custom permissions (`access_dev_tools`, `access_real_data_exports`) live on `SiteSettings`;
everything else reuses Django's auto-generated `add/change/delete/view_<model>` permissions that
Wagtail's `SnippetViewSet`s already respect.

**Tech Stack:** Django 5 / Wagtail (existing project stack). No new dependencies.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-02-admin-roles-permissions-design.md` — read it before
  starting; this plan implements only the "Dev / Data Analyst / Gig Booker / Library Manager /
  Arranger-Composer" utility-view gates and the `setup_roles` command. **The attendance/birthday
  password→login conversion (Attendance Taker, Editor image/media patch is included here but the
  password removal is not) is a separate follow-up plan** — do not touch `http_basic_auth`,
  `http_basic_auth_birthdays`, `attendance_capture`, `inactive_members`, `attendance_reports`,
  `attendance_section_report_new`, `gigs_for_date`, or `birthdays` in this plan.
- Work happens on the `admin-roles-permissions` branch (already checked out, based on
  `development`). Do not commit to `main`.
- GPG-sign all commits (`git commit -S`). No emojis, no `Co-Authored-By` lines. Conventional prefix
  (`feat:`, `test:`, `chore:`).
- `isort --check-only blowcomotion/ --diff` runs on commit via pre-commit hook — run
  `isort blowcomotion/` before committing any task that adds an import.
- Wagtail's `wagtailadmin.access_admin` permission gates entry to `/admin/` itself (Wagtail's own
  `require_admin_access` wrapper checks `request.user.has_perm('wagtailadmin.access_admin')`
  before any view registered via `register_admin_urls` runs). Every new role group whose views live
  under `/admin/` must include this permission or its members will be bounced before reaching the
  view.

```python
from django.contrib.auth.models import Group, Permission
from django.contrib.contenttypes.models import ContentType
from django.core.management.base import BaseCommand

from blowcomotion.models import (
    AttendanceRecord,
    CachedGig,
    Chart,
    CustomImage,
    Equipment,
    Event,
    InstrumentHistoryLog,
    InstrumentStorageLocation,
    LibraryInstrument,
    Song,
    SongConductor,
    SongSoloist,
    SongVideo,
)


def _model_perms(model, actions=("add", "change", "delete", "view")):
    ct = ContentType.objects.get_for_model(model)
    codenames = [f"{action}_{model._meta.model_name}" for action in actions]
    return list(Permission.objects.filter(content_type=ct, codename__in=codenames))


def _named_perm(app_label, codename):
    return list(Permission.objects.filter(content_type__app_label=app_label, codename=codename))


ACCESS_ADMIN = lambda: _named_perm("wagtailadmin", "access_admin")

ROLE_PERMISSIONS = {
    "Dev": lambda: (
        _named_perm("blowcomotion", "access_dev_tools")
        + _model_perms(CachedGig)
        + ACCESS_ADMIN()
    ),
    "Data Analyst": lambda: (
        _named_perm("blowcomotion", "access_real_data_exports") + ACCESS_ADMIN()
    ),
    "Gig Booker": lambda: _model_perms(CachedGig) + _model_perms(Event) + ACCESS_ADMIN(),
    "Library Manager": lambda: (
        _model_perms(LibraryInstrument)
        + _model_perms(InstrumentHistoryLog)
        + _model_perms(InstrumentStorageLocation)
        + _model_perms(Equipment)
        + ACCESS_ADMIN()
    ),
    "Arranger/Composer": lambda: (
        _model_perms(Chart)
        + _model_perms(Song)
        + _model_perms(SongConductor)
        + _model_perms(SongSoloist)
        + _model_perms(SongVideo)
        + ACCESS_ADMIN()
    ),
    # Attendance Taker's permission set (AttendanceRecord view/add/change) lands
    # with the attendance/birthday login-conversion follow-up plan; created here
    # so the group exists and can be assigned to users ahead of that work.
    "Attendance Taker": lambda: [],
}

EDITOR_GROUP_NAMES = ("Editors", "Moderators")


class Command(BaseCommand):
    help = "Create/update role Groups with their permission sets (safe to re-run)"

    def handle(self, *args, **options):
        for name, get_perms in ROLE_PERMISSIONS.items():
            group, created = Group.objects.get_or_create(name=name)
            group.permissions.set(get_perms())
            self.stdout.write(
                f"{'Created' if created else 'Updated'} group '{name}' "
                f"({group.permissions.count()} perms)"
            )

        self._patch_editor_groups()

    def _patch_editor_groups(self):
        image_perms = _model_perms(CustomImage)
        media_ct = ContentType.objects.filter(app_label="wagtailmedia", model="media").first()
        media_perms = list(Permission.objects.filter(content_type=media_ct)) if media_ct else []
        if not media_perms:
            self.stdout.write(self.style.WARNING("wagtailmedia.Media content type not found, skipping media perms"))

        for group_name in EDITOR_GROUP_NAMES:
            try:
                group = Group.objects.get(name=group_name)
            except Group.DoesNotExist:
                self.stdout.write(self.style.WARNING(f"Group '{group_name}' not found, skipping"))
                continue
            group.permissions.add(*image_perms, *media_perms)
            self.stdout.write(f"Patched '{group_name}' with CustomImage + Media permissions")
```

class PermissionMenuItem(MenuItem):
    def __init__(self, *args, permission=None, **kwargs):
        self._permission = permission
        super().__init__(*args, **kwargs)

    def is_shown(self, request):
        if self._permission is None:
            return True
        return request.user.has_perm(self._permission)


class PermissionSubmenuMenuItem(SubmenuMenuItem):
    def __init__(self, *args, permission=None, **kwargs):

        super().__init__(*args, **kwargs)

        if self._permission is None:
            return True
        return request.user.has_perm(self._permission)
